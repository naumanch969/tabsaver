The one-liner
Save your open tabs as a named workspace. Close them without fear. Reopen in one click.
That's it. Everything else is polish around this core loop.

Design principles
2 clicks max
Save workspace = icon click + one button. Never more. Speed is the feature.
Never destructive
Close tabs only when user explicitly says "Save & close." Never surprise-delete anything.
Always glanceable
Popup opens to your workspace list instantly. No loading, no splash screen.
Inline, no new tab
Everything happens in the popup. Don't redirect users to a settings page to do basic things.
The moments that make users stay
1
First save — "Oh, this is it"
User saves their first workspace. Sees tabs close cleanly. Immediately feels lighter. This is the aha moment. It must happen in the first 90 seconds of installing.
2
Next morning — "It remembered"
"Resume yesterday's session?" appears on browser open. User clicks yes. All their tabs are back in 2 seconds. This is when they become a daily user.
3
Crash recovery — "It saved me"
Browser crashes. User panics. Opens Tab Saver. Sees "Auto-saved 4 min ago." Restores everything. Emotional peak — they tell someone about this.
4
Share moment — "Here, look at this"
User shares a workspace link with a colleague. Colleague installs the extension to open it. One organic install from a single share. This is your growth loop.
5
Stale nudge — "Oh right, I forgot"
User sees "Weekend reads · 18 days old." Clicks through, reads one article, deletes two dead links. Workspace inbox-zero. Keeps them engaged past the first week.

What NOT to build in v1
No tags or folders. No search. No team sharing. No browser history. No AI. No settings page with 20 options. Complexity kills the aha moment — one clean loop first.
